# Minmes
Концепт частично децентрализованного мессенджера с очень странным транспортом для обхода цензуры

# Особенности
Это мессенджер, гоняющий сообщения через публичные MQTT и XMPP сервера, попутно шифруя все сообщения.

# Текущее состояние
В данный момент это только зарождение проекта, я далеко не лучший программист, но надеюсь довести проект до рабочего состояния

# Как оно всё будет работать
MQTT: в клиенте будут встроенные MQTT брокеры, будет возможность их добавлять/удалять. При попытке отправки сообщения клиент будет по очереди писать в каждый доступный mqtt брокер по пути minmes/{pubkey}/inbox сообщение-пинг. Клиент с другой стороны ответит сообщение-пинг. Оба клиента запомнят этого брокера как рабочего и предпочитаемого, затем просто будут слать зашифрованные сообщения по этому топику. Если один из клиентов не сможет достучаться до брокера, то он попытается написать адресату через других брокеров, сначала отправив сообщения-пинг на все доступные каналы.
Передача контактов планируется лишь по QR кодам. В QR коде будет содержаться AES ключ + pubkey. Шифрованием будет AES256, нерушимая классика как никак.
